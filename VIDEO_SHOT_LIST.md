## Technical Specifications

### Recording Settings
- **Resolution**: 1920x1080 (Full HD) minimum, 2560x1440 preferred
- **Frame Rate**: 30fps minimum, 60fps preferred for smooth UI
- **Codec**: H.264 or H.265
- **Bitrate**: 10-15 Mbps for web, 5-8 Mbps for social media

### Browser Settings
- **Browser**: Chrome or Firefox (latest version)
- **Window Size**: 1920x1080
- **Zoom Level**: 100% (no browser zoom)
- **Extensions**: Disable all extensions that might interfere
- **Theme**: Use light theme for better visibility

### UI Settings
- **Theme**: Light mode (better for video)
- **Sidebar**: Expanded (not collapsed)
- **Font Size**: Default (ensure text is readable)

### Cursor Settings
- **Cursor Style**: Large, visible cursor
- **Cursor Speed**: Medium (not too fast)
- **Highlight Clicks**: Enable click highlighting if available
- **Smooth Movements**: Avoid jerky mouse movements

### Audio Settings
- **Microphone**: High-quality microphone (USB or XLR)
- **Sample Rate**: 48kHz
- **Bit Depth**: 24-bit
- **Format**: WAV for recording, MP3 for final export
- **Noise Reduction**: Apply in post-production

## Notes for Director/Producer

1. **Test All Interactions First**: Record a test run to ensure all UI elements work as expected
2. **Have Backup Data**: Prepare backup locations/scenarios in case primary doesn't work
3. **Record Multiple Takes**: Record each scene 2-3 times to have options
4. **Keep Raw Footage**: Save raw recordings before editing
5. **Document Issues**: Note any bugs or UI issues encountered during recording
6. **Timing**: Allow extra time for loading states and transitions
7. **Pacing**: Don't rush—allow viewers to absorb information
8. **Consistency**: Use same location (Queensgate) throughout for narrative coherence
